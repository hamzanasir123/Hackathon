## Domain Randomization and Synthetic Data

### Understanding Domain Randomization

<CodeExample
  language="python"
  title="Domain Randomization Example"
  description="Python code to configure domain randomization in Isaac Sim"
  code={`import omni
from omni.isaac.core.utils.prims import get_prim_at_path
from omni.isaac.core.utils.stage import add_reference_to_stage

# Configure domain randomization parameters
def configure_domain_randomization():
    # Randomize lighting conditions
    light_prim = get_prim_at_path("/World/Light")
    light_prim.GetAttribute("inputs:intensity").Set(100 + random.uniform(-50, 50))

    # Randomize material properties
    material_prim = get_prim_at_path("/World/Materials/RandomMaterial")
    color = (random.random(), random.random(), random.random())
    material_prim.GetAttribute("inputs:diffuse_color_constant").Set(color)

    # Randomize object positions
    for i in range(10):
        obj_prim = get_prim_at_path(f"/World/Object_{i}")
        new_pos = [random.uniform(-2, 2), random.uniform(-2, 2), 0.1]
        obj_prim.GetAttribute("xformOp:translate").Set(new_pos)`}
/>

Domain randomization is crucial for creating robust perception systems that can transfer from simulation to reality.

## Isaac Sim Workflows and Best Practices

### Basic Isaac Sim Workflow

1. **Scene Setup**: Create or load a USD scene with robot and environment
2. **Sensor Configuration**: Add and configure sensors (LiDAR, cameras, IMU)
3. **Physics Setup**: Configure collision properties and dynamics
4. **Simulation Execution**: Run the simulation and collect data
5. **Data Processing**: Process simulation data for training or validation

<CodeExample
  language="python"
  title="Basic Isaac Sim Setup"
  description="Python code to set up a basic Isaac Sim environment"
  code={`import omni
from omni.isaac.core import World
from omni.isaac.core.utils.nucleus import get_assets_root_path
from omni.isaac.core.utils.stage import add_reference_to_stage

# Create a simulation world
my_world = World(stage_units_in_meters=1.0)

# Add a robot to the scene
assets_root_path = get_assets_root_path()
if assets_root_path is None:
    print("Could not use Isaac Sim assets. Ensure Isaac Sim is installed.")
else:
    # Load a robot asset
    add_reference_to_stage(
        usd_path=assets_root_path + "/Isaac/Robots/Franka/franka.usd",
        prim_path="/World/Robot"
    )

    # Add sensors to the robot
    my_world.add_sensor("camera", "/World/Robot/base_link")
    my_world.add_sensor("lidar", "/World/Robot/base_link")

    # Reset and step the world
    my_world.reset()
    for i in range(1000):
        my_world.step(render=True)`}
/>

## Isaac ROS Integration

Isaac ROS provides hardware-accelerated perception and navigation packages that work seamlessly with Isaac Sim:

<CodeExample
  language="python"
  title="Isaac ROS Node Example"
  description="Example of an Isaac ROS node that processes sensor data"
  code={`#!/usr/bin/env python3
import rclpy
from rclpy.node import Node
from sensor_msgs.msg import Image
from isaac_ros_messages.msg import FeatureArray

class IsaacPerceptionNode(Node):
    def __init__(self):
        super().__init__('isaac_perception_node')

        # Subscribe to camera image
        self.image_sub = self.create_subscription(
            Image,
            '/camera/image_raw',
            self.image_callback,
            10
        )

        # Publish features
        self.feature_pub = self.create_publisher(
            FeatureArray,
            '/features',
            10
        )

    def image_callback(self, msg):
        # Process image using Isaac ROS GEMs
        features = self.extract_features(msg)
        self.feature_pub.publish(features)`}
